# 🤝 Contributing to ITIC Learning Resources

First off, thank you for considering contributing! This repository is built *by* ITIC members *for* ITIC members (and anyone else learning tech), and it only grows through community contributions.

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

---

## 📌 Ways to Contribute

You don't need to be an expert to contribute. Useful contributions include:

- ➕ Adding a new tutorial, article, video, or course link to a track folder
- 📝 Writing your own notes, cheat sheet, or roadmap
- 🐛 Fixing broken links or outdated information
- 🗂 Improving the organization of an existing folder
- 🌐 Translating resources
- ✍️ Fixing typos or improving clarity

---

## 🚀 Getting Started

1. **Fork** this repository.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/<your-username>/learning-resources.git
   cd learning-resources
   ```
3. **Create a new branch** for your change:
   ```bash
   git checkout -b add-react-hooks-guide
   ```
4. Make your changes (see [Guidelines](#-content-guidelines) below).
5. **Commit** with a clear message:
   ```bash
   git commit -m "Add: React Hooks guide to web-development track"
   ```
6. **Push** to your fork:
   ```bash
   git push origin add-react-hooks-guide
   ```
7. Open a **Pull Request** against the `main` branch of this repository.

---

## 📁 Content Guidelines

- **Place resources in the correct track folder.** If a suitable folder doesn't exist yet, propose a new one in your PR description.
- **Update the folder's `README.md` index** when adding a resource, using this format:
  ```markdown
  - [Resource Title](https://link-to-resource) — one-line description of what it covers
  ```
- **Prefer free, high-quality, and up-to-date resources.** Paid resources are acceptable only if clearly labeled `(paid)`.
- **No plagiarism.** Only add original notes/cheat sheets you wrote yourself, or link out to the original source — never copy-paste large chunks of copyrighted material.
- **Keep filenames lowercase and hyphenated**, e.g. `git-basics-cheatsheet.md`.
- **One topic per pull request** where possible — this makes reviews faster and easier.

---

## ✅ Pull Request Checklist

Before submitting, please confirm:

- [ ] My change is in the correct track folder
- [ ] I updated the relevant `README.md` index
- [ ] Links work and point to reputable sources
- [ ] My commit messages are clear
- [ ] I have read and agree to the [Code of Conduct](CODE_OF_CONDUCT.md)

---

## 🔍 Review Process

- A maintainer will review your PR, usually within a few days.
- You may be asked to make small changes before merging.
- Once approved, your contribution will be merged and you'll be credited automatically via our contributors widget on the main README.
