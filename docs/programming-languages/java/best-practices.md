# 🏆 Java Best Practices & Clean Code

Write readable, maintainable, and idiomatic Java code.

---

## 💡 Top Principles
1. **SOLID Principles**: Single Responsibility, Open-Closed, Liskov Substitution, Interface Segregation, Dependency Inversion.
2. **Proper Naming**: Variables/Methods in `camelCase`, Classes in `PascalCase`, constants in `UPPER_SNAKE_CASE`.
3. **Use Collections Wisely**: Program against interfaces (e.g. `List<String> list = new ArrayList<>()`) instead of concrete implementations.
4. **Avoid NullPointerExceptions**: Use Java 8's `Optional<T>` API or check variables for null.
